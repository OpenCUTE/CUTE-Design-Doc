# 精度格式总览

## 1. 术语说明

| 术语 | 说明 |
|------|------|
| ElementDataType | CUTE 数据类型枚举，4-bit 编码（`DataTypeBitWidth = 4`），定义 13 种类型 |
| ReduceWidthByte | 每个 PE 的归约通道字节宽度（默认 32） |
| ScaleElementWidth | 缩放因子单个元素位宽（8 bit，E5M2 格式） |
| MinDataTypeWidth | PE 支持的最小数据类型位宽（4 bit，即 FP4） |
| MinGroupSize | 最小块缩放分组大小（16 个元素） |

## 2. 设计规格

| 属性 | 说明 |
|------|------|
| 编码位宽 | 4 bit |
| 已实现类型数 | 13 种 |
| 累加精度 | FP32（浮点输入）或 INT32（整数输入） |
| 块缩放类型 | 4 种（MXFP8 E4M3、MXFP8 E5M2、NVFP4、MXFP4） |

## 3. 功能描述

### 3.1 整数类型

**INT8（8-bit 有符号整数）：**

| 属性 | 值 |
|------|-----|
| 总位宽 | 8 bit |
| 符号位 | 1 bit |
| 数值范围 | -128 ~ +127 |
| 累加类型 | INT32 |

**UINT8（8-bit 无符号整数）：** 范围 0 ~ 255，支持与 INT8 的混合运算。

CUTE 支持四种整数类型组合：

| 编码 | 源码名称 | A 类型 | B 类型 | 说明 |
|------|---------|--------|--------|------|
| 0 | `DataTypeI8I8I32` | INT8 | INT8 | 对称量化 |
| 4 | `DataTypeI8U8I32` | INT8 | UINT8 | 混合符号 |
| 5 | `DataTypeU8I8I32` | UINT8 | INT8 | 混合符号 |
| 6 | `DataTypeU8U8I32` | UINT8 | UINT8 | 无符号量化 |

### 3.2 标准浮点类型

**FP16（IEEE 754 半精度）：**

```
┌───┬─────┬──────────────────────────────┐
│ S │ Exp │        Mantissa             │
│ 1 │  5  │           10                │
└───┴─────┴──────────────────────────────┘
```

| 属性 | 值 |
|------|-----|
| 编码 | 1 (`DataTypeF16F16F32`) |
| 总位宽 | 16 bit |
| 指数位 | 5 bit（bias=15） |
| 尾数位 | 10 bit |
| 数值范围 | ±65504 |

**BF16（Brain Float）：**

```
┌───┬─────┬──────────────────────────────┐
│ S │ Exp │        Mantissa             │
│ 1 │  8  │            7                │
└───┴─────┴──────────────────────────────┘
```

| 属性 | 值 |
|------|-----|
| 编码 | 2 (`DataTypeBF16BF16F32`) |
| 总位宽 | 16 bit |
| 指数位 | 8 bit（bias=127，与 FP32 相同） |
| 尾数位 | 7 bit |
| 数值范围 | ±3.4×10³⁸（与 FP32 相同） |

BF16 保留了 FP32 的指数范围，适合深度学习训练中梯度范围较大的场景。

**TF32（TensorFloat-32）：**

```
┌───┬─────┬──────────────┬─────────────────────────┐
│ S │ Exp │   Mantissa   │       (ignored)         │
│ 1 │  8  │     10       │          13             │
└───┴─────┴──────────────┴─────────────────────────┘
```

| 属性 | 值 |
|------|-----|
| 编码 | 3 (`DataTypeTF32TF32F32`) |
| 总位宽 | 32 bit（19 bit 有效） |
| 指数位 | 8 bit |
| 有效尾数位 | 10 bit |
| 数值范围 | ±3.4×10³⁸ |

TF32 在 FP32 的 32 位容器中使用 19 位有效数据，可复用 FP32 数据通路。

### 3.3 低精度浮点类型（标准 FP8）

**FP8 E4M3：**

```
┌───┬─────┬───────────┐
│ S │ Exp │ Mantissa  │
│ 1 │  4  │     3     │
└───┴─────┴───────────┘
```

| 属性 | 值 |
|------|-----|
| 编码 | 11 (`DataTypefp8e4m3F32`) |
| 总位宽 | 8 bit |
| 指数位 | 4 bit（bias=7） |
| 尾数位 | 3 bit |
| 数值范围 | ±448 |
| 需要 Scale | 否 |

E4M3 格式没有 NaN/Inf 编码，适合推理前向传播。

**FP8 E5M2：**

```
┌───┬─────┬───────────┐
│ S │ Exp │ Mantissa  │
│ 1 │  5  │     2     │
└───┴─────┴───────────┘
```

| 属性 | 值 |
|------|-----|
| 编码 | 12 (`DataTypefp8e5m2F32`) |
| 总位宽 | 8 bit |
| 指数位 | 5 bit（bias=15） |
| 尾数位 | 2 bit |
| 数值范围 | ±57344 |
| 需要 Scale | 否 |

E5M2 格式支持 NaN/Inf 编码，适合梯度计算。

### 3.4 块缩放浮点类型（Microscaling）

块缩放类型在标准低精度浮点的基础上引入共享缩放因子，每个缩放因子覆盖一组连续元素。

**MXFP8 E4M3：**

| 属性 | 值 |
|------|-----|
| 编码 | 7 (`DataTypeMxfp8e4m3F32`) |
| 元素位宽 | 8 bit（E4M3） |
| 块大小 | 32 个元素 |
| 缩放因子位宽 | 8 bit（E5M2 无符号） |
| 每 PE 每周期 Scale 宽度 | `ReduceWidth × 8 / 8 / 32` bit |

**MXFP8 E5M2：**

| 属性 | 值 |
|------|-----|
| 编码 | 8 (`DataTypeMxfp8e5m2F32`) |
| 元素位宽 | 8 bit（E5M2） |
| 块大小 | 32 个元素 |
| 缩放因子位宽 | 8 bit（E5M2 无符号） |
| 每 PE 每周期 Scale 宽度 | `ReduceWidth × 8 / 8 / 32` bit |

**MXFP4：**

| 属性 | 值 |
|------|-----|
| 编码 | 10 (`DataTypemxfp4F32`) |
| 元素位宽 | 4 bit |
| 块大小 | 32 个元素 |
| 缩放因子位宽 | 8 bit（E5M2 无符号） |
| 每 PE 每周期 Scale 宽度 | `ReduceWidth × 8 / 4 / 32` bit |

**NVFP4：**

| 属性 | 值 |
|------|-----|
| 编码 | 9 (`DataTypenvfp4F32`) |
| 元素位宽 | 4 bit |
| 块大小 | 16 个元素 |
| 缩放因子位宽 | 8 bit（E5M2 无符号） |
| 每 PE 每周期 Scale 宽度 | `ReduceWidth × 8 / 4 / 16` bit |

**ScaleWidth 通用公式：**

```
ScaleWidth = ReduceWidthByte × 8 × ScaleElementWidth / MinDataTypeWidth / MinGroupSize
           = ReduceWidthByte × 8 × 8 / 4 / 16
           = ReduceWidthByte（即与 ReduceWidthByte 相等的字节数）
```

## 5. 硬件解码路径

FReducePE 中的 `FVecDecoder` 根据 4-bit `dataType` 字段将不同输入类型统一解码为内部 TF32 表示：

```
dataType ──→ FVecDecoder ──→ 内部 TF32 格式

0/4/5/6 (I8/U8)    ──→ 零扩展 ──→ 整数乘法路径
1 (FP16)           ──→ 拆分 S/E5/M10 ──→ 尾数映射到 TF32
2 (BF16)           ──→ 扩展 E8/M7 ──→ 尾数映射到 TF32
3 (TF32)           ──→ 直接使用 E8/M10
7/8 (MXFP8)        ──→ 扩展指数和尾数 + Scale 缩放
9 (NVFP4)          ──→ 扩展指数和尾数 + Scale 缩放
10 (MXFP4)         ──→ 扩展指数和尾数 + Scale 缩放
11 (FP8 E4M3)      ──→ 扩展指数和尾数
12 (FP8 E5M2)      ──→ 扩展指数和尾数
```

块缩放类型（7/8/9/10）在 FReducePE 外部由 ScaleController 提供 `ScaleVecWidth` 宽度的缩放因子向量，与数据向量同步传入 MTE。
